# Savings Estimation Methods

## Purpose

This document defines how to calculate monthly and annual cost savings for every recommendation type generated by the subagents. Accurate savings estimation is critical for prioritization and stakeholder buy-in. All pricing data is sourced from the Azure Retail Prices API.

---

## Azure Retail Prices API

### Endpoint

```
https://prices.azure.com/api/retail/prices
```

This is a public, unauthenticated API. No Azure credentials are required.

### Base Query Parameters

| Parameter | Description |
|---|---|
| `$filter` | OData filter expression for pricing criteria |
| `currencyCode` | Currency (default: `USD`) |
| `api-version` | API version (use `2023-01-01-preview` or latest) |

### Common Filter Syntax for VMs

```
armRegionName eq '{region}'
and armSkuName eq '{skuName}'
and priceType eq 'Consumption'
and serviceName eq 'Virtual Machines'
```

### OS-Specific Pricing

VM pricing differs by OS. Use the `meterName` field to distinguish:

| OS | meterName Filter | Example |
|---|---|---|
| Linux | Does NOT contain `Windows` | `meterName` = `D4s v3` |
| Windows | Contains `Windows` | `meterName` = `D4s v3 Windows` |

**Full API call example for a Linux D4s_v3 in East US:**

```
GET https://prices.azure.com/api/retail/prices?$filter=armRegionName eq 'eastus' and armSkuName eq 'Standard_D4s_v3' and priceType eq 'Consumption' and serviceName eq 'Virtual Machines'
```

Then filter the response items where `meterName` does NOT contain "Windows" for Linux pricing.

### API Pagination

The API returns results in pages. Always handle pagination:

```python
url = "https://prices.azure.com/api/retail/prices?$filter=..."
all_items = []

while url:
    response = requests.get(url).json()
    all_items.extend(response.get("Items", []))
    url = response.get("NextPageLink")  # None when no more pages
```

### Caching

- Pricing data may be cached for the duration of a single scan run.
- Do NOT cache across scan runs — prices change regularly.
- Cache key: `{armSkuName}:{armRegionName}:{osType}:{priceType}`

## Monthly Cost Formula

All cost calculations use a standard month of **730 hours** (365 days × 24 hours ÷ 12 months):

```
Monthly Cost = Hourly Rate × 730
```

### Rounding

All monetary values must be rounded to **2 decimal places** (USD cents).

```python
monthly_cost = round(hourly_rate * 730, 2)
```

## Savings Formulas by Recommendation Type

### 1. VM Rightsizing (Resize)

```
Monthly Savings = Current Monthly Cost - Target Monthly Cost
Annual Savings = Monthly Savings × 12
```

**Steps:**
1. Look up current SKU hourly price via Retail Prices API
2. Look up target SKU hourly price via Retail Prices API
3. Ensure both use the same OS type (Linux/Windows) and region
4. Calculate: `(current_hourly - target_hourly) × 730`

**Example:**
```
Current: Standard_D8s_v3 Linux in eastus = $0.384/hr → $280.32/mo
Target:  Standard_D4s_v3 Linux in eastus = $0.192/hr → $140.16/mo
Monthly Savings: $280.32 - $140.16 = $140.16
Annual Savings: $140.16 × 12 = $1,681.92
```

### 2. Deallocated VMs (Delete)

For VMs that have been deallocated for more than the threshold (default 30 days):

```
Monthly Savings = Current VM Monthly Cost (compute only, since deallocated VMs don't incur compute charges)
```

**Important nuance:** Deallocated VMs do NOT incur compute charges, but they DO incur:
- OS disk costs
- Data disk costs
- Public IP costs (if static)
- Any associated reserved capacity waste

For deallocated VMs, savings = **disk + IP costs only** (compute is already $0):

```
Monthly Savings = Sum of attached disk monthly costs + static public IP cost
```

But the recommendation to **delete** also recovers any reserved instance waste, so flag this separately.

### 3. Stopped-Not-Deallocated VMs (Deallocate or Delete)

**This is a Critical finding.** VMs in the "Stopped" state (not "Deallocated") continue to incur full compute charges.

```
Monthly Savings = Full VM Compute Monthly Cost
```

This is the full hourly rate × 730, because the VM is billing as if running.

### 4. Unattached Managed Disks (Delete)

Disks not attached to any VM (where `managedBy` is null):

```
Monthly Savings = Disk Monthly Cost based on SKU tier and size
```

Disk pricing varies by:
- SKU tier: Premium_LRS, StandardSSD_LRS, Standard_LRS, UltraSSD_LRS
- Disk size: P10, P20, P30, etc. (for Premium); E10, E20, E30, etc. (for StandardSSD)

**API filter for disk pricing:**
```
serviceName eq 'Storage' and armRegionName eq '{region}' and meterName eq '{diskSizeTier}'
```

### 5. Aged Snapshots (Delete)

Snapshots older than the threshold (default 90 days):

```
Monthly Savings = Snapshot Size (GB) × Per-GB Snapshot Rate
```

Snapshot pricing is per GB stored, varying by region and redundancy.

### 6. Unused Load Balancers (Delete)

Load balancers with zero backend pool members:

```
Monthly Savings = LB Hourly Rate × 730
```

**Standard Load Balancer pricing** (as of 2025):
- Approximately $0.025/hr for the first 5 rules, plus per-rule charges

**API filter:**
```
serviceName eq 'Load Balancer' and armRegionName eq '{region}' and skuName eq 'Standard'
```

### 7. Unused Application Gateways (Delete)

Application gateways with zero backend targets:

```
Monthly Savings = AppGW Hourly Rate × 730
```

**API filter:**
```
serviceName eq 'Application Gateway' and armRegionName eq '{region}'
```

Note: Application Gateway pricing is complex (fixed + capacity units). Use the fixed component as minimum savings.

### 8. Unassociated Public IPs (Delete)

Public IPs not associated with any resource:

```
Monthly Savings = Public IP Hourly Rate × 730
```

Standard static public IPs incur charges when unassociated. Basic SKU public IPs do NOT incur charges when unassociated.

### 9. Empty App Service Plans (Delete or Downgrade)

App Service Plans with zero apps deployed:

```
Monthly Savings = App Service Plan Monthly Cost
```

**API filter:**
```
serviceName eq 'Azure App Service' and armRegionName eq '{region}' and armSkuName eq '{tier}'
```

### 10. App Service Plan Rightsizing (Resize)

```
Monthly Savings = Current Plan Monthly Cost - Target Plan Monthly Cost
```

### 11. SQL Database Rightsizing (Resize)

```
Monthly Savings = Current SQL Tier Monthly Cost - Target SQL Tier Monthly Cost
```

For DTU-based: pricing by DTU level (S0, S1, S2, etc.)
For vCore-based: pricing by vCore count and tier (General Purpose, Business Critical)

### 12. Storage Account Tier Change (Hot → Cool/Archive)

```
Monthly Savings = (Hot per-GB rate - Cool per-GB rate) × Storage Size GB
```

Note: Cool tier has higher access costs, so only recommend when access is infrequent.

## Currency Handling

- **Default currency:** USD
- If the subscription uses a different billing currency, note this in the recommendation but still calculate in USD for consistency.
- The Retail Prices API supports `currencyCode` parameter for non-USD queries if needed.
- All internal calculations and comparisons should use USD.

---

## Pricing Lookup Decision Tree

```
1. Identify resource type and current SKU
2. Determine region (from resource ARM ID)
3. Determine OS type (for VMs: Linux or Windows)
4. Query Retail Prices API with appropriate filters
5. Handle pagination (follow NextPageLink)
6. Filter results by meterName for OS-specific pricing
7. Select the 'Consumption' pay-as-you-go rate (not Reservation or Spot)
8. Calculate monthly cost = hourly rate × 730
9. Round to 2 decimal places
```

---

## Error Handling

| Scenario | Action |
|---|---|
| SKU not found in Retail Prices API | Log warning, set savings to $0, set `dataQuality` to "Partial" |
| Region not found | Try with canonical region name (e.g., `eastus` not `East US`) |
| Multiple matching prices returned | Select the one with `type` = "Consumption" and `isPrimaryMeterRegion` = true |
| API rate limiting (429) | Retry with exponential backoff (1s, 2s, 4s, max 30s) |
| API unavailable | Log error, skip savings calculation, set savings to $0 |

---

## Validation Rules

Before emitting a savings estimate:

1. `monthlySavings` must be ≥ 0 (never recommend a more expensive SKU)
2. `annualSavings` must equal `monthlySavings × 12`
3. `recommendation.monthlyCost` must be ≤ `currentState.monthlyCost`
4. If savings is $0 but a resize is recommended, log a warning — this likely indicates a pricing lookup failure
5. Cross-check: if Azure Advisor also has a savings estimate for the same resource, compare. Log if delta > 20%.
