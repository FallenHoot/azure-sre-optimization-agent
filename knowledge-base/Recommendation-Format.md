# Recommendation Output Format

## Purpose

This document defines the standard output schema for all optimization recommendations generated by any subagent (Compute, Storage, Network, PaaS, Governance). All subagents **must** emit recommendations conforming to this schema to enable consistent aggregation, reporting, and ticket creation by the orchestrator.

---

## Recommendation JSON Schema

```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "title": "OptimizationRecommendation",
  "type": "object",
  "required": [
    "id",
    "timestamp",
    "category",
    "subcategory",
    "severity",
    "resourceId",
    "resourceName",
    "resourceGroup",
    "subscription",
    "currentState",
    "recommendation",
    "fitScore",
    "evidence",
    "riskAssessment"
  ],
  "properties": {
    "id": {
      "type": "string",
      "description": "Unique recommendation ID. Format: {category}-{subscriptionId short}-{resourceName}-{timestamp}. Example: compute-a1b2c3-myvm01-20260213T140000Z"
    },
    "timestamp": {
      "type": "string",
      "format": "date-time",
      "description": "ISO 8601 UTC timestamp when the recommendation was generated."
    },
    "category": {
      "type": "string",
      "enum": ["Compute", "Storage", "Network", "PaaS", "Governance"],
      "description": "Primary category matching the subagent that generated the recommendation."
    },
    "subcategory": {
      "type": "string",
      "description": "Specific recommendation type within the category.",
      "examples": [
        "VM Rightsizing",
        "Deallocated VM Cleanup",
        "Stopped-Not-Deallocated VM",
        "Unattached Disk",
        "Aged Snapshot",
        "Unused Load Balancer",
        "Unused Application Gateway",
        "Unassociated Public IP",
        "App Service Plan Rightsizing",
        "SQL DB Rightsizing",
        "Credential Expiry",
        "Outdated API Version"
      ]
    },
    "severity": {
      "type": "string",
      "enum": ["Critical", "High", "Medium", "Low"],
      "description": "Severity level per Severity-Classification.md rules."
    },
    "resourceId": {
      "type": "string",
      "description": "Full Azure Resource Manager resource ID."
    },
    "resourceName": {
      "type": "string",
      "description": "Short resource name (last segment of resourceId)."
    },
    "resourceGroup": {
      "type": "string",
      "description": "Resource group name containing the resource."
    },
    "subscription": {
      "type": "object",
      "properties": {
        "subscriptionId": {
          "type": "string",
          "description": "Azure subscription GUID."
        },
        "subscriptionName": {
          "type": "string",
          "description": "Human-readable subscription display name."
        }
      },
      "required": ["subscriptionId", "subscriptionName"]
    },
    "currentState": {
      "type": "object",
      "properties": {
        "sku": {
          "type": "string",
          "description": "Current SKU or tier of the resource. E.g., Standard_D4s_v3, Premium_LRS P30."
        },
        "monthlyCost": {
          "type": "number",
          "description": "Estimated current monthly cost in USD, rounded to 2 decimal places."
        },
        "properties": {
          "type": "object",
          "description": "Additional resource-specific properties (e.g., vCPUs, memoryGB, diskSizeGB, powerState).",
          "additionalProperties": true
        }
      },
      "required": ["sku", "monthlyCost"]
    },
    "recommendation": {
      "type": "object",
      "properties": {
        "action": {
          "type": "string",
          "description": "Recommended action verb.",
          "examples": [
            "Resize",
            "Delete",
            "Deallocate",
            "Downgrade",
            "ChangeTier",
            "RotateCredential",
            "UpdateAPIVersion",
            "EnableLifecyclePolicy"
          ]
        },
        "targetSku": {
          "type": ["string", "null"],
          "description": "Target SKU after action. Null for Delete/Deallocate actions."
        },
        "monthlyCost": {
          "type": "number",
          "description": "Estimated monthly cost after action. 0 for Delete actions."
        },
        "monthlySavings": {
          "type": "number",
          "description": "Monthly savings = currentState.monthlyCost - recommendation.monthlyCost. Rounded to 2 decimal places."
        },
        "annualSavings": {
          "type": "number",
          "description": "Annual savings = monthlySavings × 12. Rounded to 2 decimal places."
        },
        "description": {
          "type": "string",
          "description": "Human-readable description of the recommendation and its rationale."
        }
      },
      "required": ["action", "targetSku", "monthlyCost", "monthlySavings", "annualSavings", "description"]
    },
    "fitScore": {
      "type": "object",
      "properties": {
        "score": {
          "type": "number",
          "minimum": 1,
          "maximum": 5,
          "description": "Overall FitScore from 1 (poor fit) to 5 (excellent fit). See FitScore-Methodology.md."
        },
        "breakdown": {
          "type": "object",
          "properties": {
            "cpuScore": {
              "type": "number",
              "description": "CPU adequacy sub-score (1-5)."
            },
            "memoryScore": {
              "type": "number",
              "description": "Memory adequacy sub-score (1-5)."
            },
            "diskScore": {
              "type": "number",
              "description": "Disk IOPS/throughput adequacy sub-score (1-5)."
            },
            "networkScore": {
              "type": "number",
              "description": "Network bandwidth adequacy sub-score (1-5)."
            },
            "constraintsPassed": {
              "type": "boolean",
              "description": "Whether all hard constraints (accelerated networking, GPU, disk count, etc.) are met."
            },
            "constraintViolations": {
              "type": "array",
              "items": { "type": "string" },
              "description": "List of hard constraint violations, if any."
            }
          }
        }
      },
      "required": ["score", "breakdown"]
    },
    "evidence": {
      "type": "object",
      "properties": {
        "metricsLookback": {
          "type": "string",
          "description": "Lookback period used for metric analysis. E.g., '7d', '30d'."
        },
        "metricsTimeGrain": {
          "type": "string",
          "description": "Time grain of metric data. E.g., 'PT1H'."
        },
        "metrics": {
          "type": "object",
          "description": "Key metric values that drove the recommendation.",
          "properties": {
            "cpuP99": { "type": ["number", "null"] },
            "memoryP99": { "type": ["number", "null"] },
            "networkP99Mbps": { "type": ["number", "null"] },
            "diskReadIOPSP99": { "type": ["number", "null"] },
            "diskWriteIOPSP99": { "type": ["number", "null"] },
            "diskReadThroughputP99MBps": { "type": ["number", "null"] },
            "diskWriteThroughputP99MBps": { "type": ["number", "null"] }
          }
        },
        "advisorRecommendationId": {
          "type": ["string", "null"],
          "description": "Azure Advisor recommendation ID if this finding was corroborated by Advisor."
        },
        "dataQuality": {
          "type": "string",
          "enum": ["Full", "Partial", "Minimal", "None"],
          "description": "Quality of metric data. Full = all metrics available for full lookback. Partial = some metrics missing. Minimal = only basic metrics. None = no metric data (Advisor-only)."
        },
        "workloadPattern": {
          "type": ["string", "null"],
          "enum": ["SteadyState", "Burst", "Batch", "Cyclical", "Growing", null],
          "description": "Detected workload pattern if pattern analysis was performed. See Workload-Patterns.md."
        }
      },
      "required": ["metricsLookback", "metricsTimeGrain", "dataQuality"]
    },
    "riskAssessment": {
      "type": "string",
      "description": "Human-readable risk assessment. E.g., 'Low risk — VM has been deallocated for 45 days with no dependent resources.' or 'Medium risk — resizing will require a reboot; VM is in an availability set.'"
    },
    "tags": {
      "type": "object",
      "description": "Relevant Azure tags from the resource, for context.",
      "additionalProperties": true
    }
  }
}
```

## Email Report Template Structure

Weekly optimization reports sent via email must follow this structure:

### 1. Executive Summary

```
Subject: Azure Optimization Report — Week of {startDate} to {endDate}

Total Resources Scanned: {count}
Total Findings: {count}
  Critical: {count}
  High: {count}
  Medium: {count}
  Low: {count}

Total Monthly Savings Identified: ${amount}
Total Annual Savings Identified: ${amount}

Trend vs Previous Week:
  Findings: {+/-count} ({percentage}% change)
  Savings: {+/-$amount} ({percentage}% change)
```

### 2. By-Category Breakdown

| Category | Findings | Monthly Savings | Top Issue |
|---|---|---|---|
| Compute | {count} | ${amount} | {description} |
| Storage | {count} | ${amount} | {description} |
| Network | {count} | ${amount} | {description} |
| PaaS | {count} | ${amount} | {description} |
| Governance | {count} | ${amount} | N/A |

### 3. Top 10 by Savings

A table of the 10 recommendations with the highest `monthlySavings`, regardless of category:

| # | Resource | Category | Current SKU | Recommended | Monthly Savings | Severity |
|---|---|---|---|---|---|---|
| 1 | {name} | {category} | {sku} | {action + target} | ${amount} | {severity} |
| ... | ... | ... | ... | ... | ... | ... |

### 4. Critical Findings

All `Critical` severity findings listed with full details. These require immediate attention.

### 5. Trend vs Previous Week

- New findings this week (not present last week)
- Resolved findings (present last week, not this week)
- Persistent findings (present both weeks)
- Savings trend chart data points

## Grouping Rules

When presenting recommendations in reports or APIs, group them in this order:

1. **By Resource Group** — group all findings for the same resource group together
2. **Within each Resource Group, by Severity** — Critical first, then High, Medium, Low
3. **Within each Severity, by Savings** — highest `monthlySavings` first

This grouping allows teams to see all findings for their resource group together, prioritized by urgency and impact.

## Versioning

This schema is version `1.0`. Any breaking changes will increment the major version. The `schemaVersion` field should be included in all recommendation outputs:

```json
{
  "schemaVersion": "1.0",
  "recommendations": [ ... ]
}
```
